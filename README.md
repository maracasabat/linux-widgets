# Claude Usage Limits — десклет для Cinnamon (Linux Mint)

Віджет для робочого столу, який показує використання лімітів вашої підписки
Claude (Pro/Max):

- **Сесія (5 год)** — відсоток використання поточного 5-годинного вікна
- **Тиждень · всі моделі** — тижневий ліміт по всіх моделях
- **Тиждень · <модель>** — тижневий ліміт по окремій моделі (якщо є)
- **Додаткове використання** — якщо ввімкнене в підписці

Для кожного ліміту відображається прогрес-бар з кольоровою індикацією та час
скидання. Кольори й пороги налаштовуються.

## Як це працює

Десклет читає OAuth-токен із `~/.claude/.credentials.json` (створюється
Claude Code після входу) і опитує той самий ендпоінт, що й команда `/usage`
у Claude Code: `https://api.anthropic.com/api/oauth/usage`.

**Безпека:** токен нікуди не передається, крім API Anthropic. Десклет нічого
не записує і не змінює. Токен не зберігається в репозиторії — лише читається
локально під час запиту.

## Вимоги

- Linux Mint / Cinnamon 5.4+ (перевірено на Cinnamon 6.6)
- Установлений [Claude Code](https://claude.com/claude-code) з активним
  входом (підписка Pro/Max)

## Встановлення

```bash
git clone https://github.com/maracasabat/linux-widgets.git
cd linux-widgets
./install.sh
```

Скрипт створює симлінк у `~/.local/share/cinnamon/desklets/` і вмикає
десклет — він одразу з'явиться на робочому столі. Перетягніть його мишею
куди зручно. Повторний запуск `install.sh` перезавантажує код після
оновлення репозиторію.

## Використання

- **Клік** по віджету — оновити дані негайно (автоматично — кожні 5 хв)
- **Правий клік → Налаштувати…** — усі налаштування
- Якщо токен протермінувався, віджет покаже підказку — достатньо відкрити
  Claude Code, і токен оновиться автоматично

## Налаштування

Правий клік по віджету → «Налаштувати…». Три сторінки:

| Сторінка | Налаштування |
|---|---|
| **Загальні** | інтервал оновлення; джерело даних (локально / SSH); шлях до `credentials.json` |
| **Вигляд** | ширина віджета; розмір шрифту; висота прогрес-барів; заокруглення кутів; показ заголовка / часу оновлення / часу скидання; колір і непрозорість фону |
| **Кольори** | колір заголовка й тексту; кольори індикаторів (низьке / попередження / критичне); пороги попередження та критичності (типово 60 % і 85 %) |

## Віддалений акаунт через SSH

Якщо ти запускаєш Claude на іншій машині (наприклад, **VS Code Remote-SSH**),
ліміти витрачаються там, і локальний віджет їх не бачить — він читає токен
цього комп'ютера. Щоб показувати ліміти віддаленої машини, увімкни режим SSH:

1. Правий клік → «Налаштувати…» → **Загальні** → *Джерело даних* →
   «Віддалено через SSH»
2. Вкажи **SSH-адресу** — `user@host` (та сама, якою ти під'єднуєшся у VS Code)

Потрібен налаштований вхід **за ключем** (без пароля) — віджет запускає `ssh`
у `BatchMode`. На віддаленій машині має бути `python3` (для вбудованої команди).

**Чому це безпечно.** Запит виконується повністю на віддаленій машині: вона
читає свій токен і сама звертається до API Anthropic. Каналом SSH назад
повертається **лише JSON зі статистикою** — OAuth-токен ніколи не копіюється
на десктоп.

### Захищений ключ (рекомендовано)

Щоб віджет міг робити на сервері *тільки* цей запит і нічого більше, використай
**окремий** ключ (не свій основний!) і прив'яжи його до forced command. Твій
звичайний доступ (шел, VS Code Remote-SSH) при цьому не зачіпається.

**1. Окремий ключ на десктопі:**

```bash
ssh-keygen -t ed25519 -f ~/.ssh/claude_usage_widget_ed25519 -N "" -C claude-usage-widget
```

**2. Скрипт-репортер на сервері** ([server/claude-usage-report](server/claude-usage-report)):

```bash
scp server/claude-usage-report СЕРВЕР:/tmp/
ssh СЕРВЕР 'mkdir -p ~/.local/bin && mv /tmp/claude-usage-report ~/.local/bin/ && chmod 700 ~/.local/bin/claude-usage-report'
```

**3. Обмежений запис у `~/.ssh/authorized_keys` на сервері** (одним рядком;
`restrict` вимикає шел, pty та всі проброски):

```
restrict,command="/home/USER/.local/bin/claude-usage-report" ssh-ed25519 AAAA...ключ-віджета
```

**4. Окремий Host-alias на десктопі** (`~/.ssh/config`) з цим ключем:

```
Host myserver-usage
    HostName 1.2.3.4
    User USER
    IdentityFile ~/.ssh/claude_usage_widget_ed25519
    IdentitiesOnly yes
```

**5. У налаштуваннях віджета** вкажи `myserver-usage` як SSH-адресу.

Перевірка, що ключ обмежений (має повернути JSON, а не вміст файлу):

```bash
ssh myserver-usage 'cat /etc/passwd'   # → все одно лише статистика
```

> `IdentitiesOnly yes` обов'язковий — інакше ssh може запропонувати спершу твій
> основний (необмежений) ключ. Якщо використовуєш мультиплексування, дай цьому
> alias **окремий** `ControlPath`, щоб він не ділив з'єднання з інтерактивним.

## Конфігурація через змінні середовища

Для нестандартних сценаріїв (інший шлях до credentials, статичний токен,
власний ендпоінт) підтримується файл `~/.config/claude-usage-desklet/.env`
або змінні середовища сеансу Cinnamon:

| Змінна | Призначення |
|---|---|
| `CLAUDE_OAUTH_TOKEN` | OAuth-токен напряму — `credentials.json` не читається |
| `CLAUDE_CREDS_PATH` | Власний шлях до `credentials.json` |
| `CLAUDE_USAGE_URL` | Власний ендпоінт API використання |

Пріоритет: змінні процесу Cinnamon → `.env`-файл → налаштування десклета →
типові значення. Приклад — у [.env.example](.env.example):

```bash
mkdir -p ~/.config/claude-usage-desklet
cp .env.example ~/.config/claude-usage-desklet/.env
```

> **Примітка.** Токени Claude Code періодично ротуються, тож статичний
> `CLAUDE_OAUTH_TOKEN` з часом протермінується. Типовий режим (читання з
> `credentials.json`) підхоплює свіжий токен автоматично.

## Усунення несправностей

- **«Не знайдено … credentials.json»** — увійдіть у Claude Code
  (`claude` у терміналі) або вкажіть шлях у налаштуваннях / `CLAUDE_CREDS_PATH`
- **«Токен недійсний (401)»** — відкрийте Claude Code, він оновить токен;
  віджет підхопить його при наступному оновленні
- **Віджет не з'явився** — перевірте журнал: `grep claude-usage ~/.xsession-errors`;
  перезапустіть Cinnamon (`Ctrl+Alt+Esc`) або повторіть `./install.sh`
- **Зміни налаштувань не застосовуються** — з версії 1.1.1 десклет сам стежить
  за файлом налаштувань і застосовує зміни наживо (Cinnamon іноді губить
  внутрішнє DBus-сповіщення про зміну). Якщо все ж не оновилося — клацніть по
  віджету або повторіть `./install.sh`
- **SSH: `Connection refused` / `Permission denied`** — перевірте вхід за
  ключем без пароля: `ssh -o BatchMode=yes user@host echo ok`. Має вивести
  `ok` без запиту пароля
- **SSH: порожньо або помилка python** — переконайтеся, що на сервері є
  `python3`, або задайте власну команду в полі «Команда на віддаленій машині»
